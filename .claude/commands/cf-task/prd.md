# cf-task:prd

从一句话需求出发，通过结构化对话产出产品需求文档（PRD），为 `cf-task:align` 提供足够丰富的输入。

## 输入

- `/cf-task:prd "需求描述"` — 从需求描述新建 PRD
- `/cf-task:prd` — 交互式，先问用户要做什么
- `/cf-task:prd <.prd.md 路径>` — 恢复已有草稿继续讨论

## 执行步骤

### 1. 确定模式

- 如果参数是已存在的 `.prd.md` 文件路径 → **恢复模式**：Read 读取文件，展示当前内容，询问用户要调整哪些部分，进入 Step 3 继续讨论
- 如果参数是其他存在的 `.md` 文件 → 提示用户"路径不是 `.prd.md` 格式，将视为参考材料"，读取后进入 Step 3（新建模式）
- 如果参数是文本字符串 → **新建模式**：以该文本作为初始需求
- 如果无参数 → **交互模式**：询问用户"要解决什么问题？"

### 2. 快速上下文扫描

了解项目背景，为后续建议提供依据：

1. 用 Glob 检查 `.code-flow/specs/shared/_map.md` 是否存在，若有则 Read 读取（了解模板位置）
2. 如有相关设计文档或已有 PRD，用 Glob/Read 扫描了解现状
3. 询问用户确认目标用户群体和业务背景

目的：**让 PRD 基于实际业务背景**，而非泛泛的通用描述。

### 3. 交互式细化

围绕 PRD 各要素逐步推进，**每轮提 2-3 个问题**：

**背景与目标**（必选）
- 要解决什么问题？影响谁？
- 当前有什么替代方案？为什么不够用？
- 期望达成什么目标？（用数据说话，如"转化率提升5%"）

**用户与场景**（必选）
- 谁会用这个功能？用户规模多大？
- 典型使用场景是什么？

**功能需求**（必选）
- 核心功能有哪些？（P0 必做，P1 重要）
- 有什么边界情况需要处理？

**非功能需求**（按需）
- 有什么性能/安全/兼容性要求？
- 如果没有明确要求，跳过此节

**范围与边界**（必选）
- 这次做什么？明确不做的是什么？
- 有什么前提假设？

**依赖与风险**（按需）
- 依赖哪些外部因素？
- 有什么已知风险？

**交互原则**：
- 基于业务背景**提出具体默认建议**，而非空泛提问
- 每轮回答后更新理解，追问细节或推进到下一个维度
- 用户说"你定"或"随意"时，选择最合理的默认方案，明确告知用户
- 输出语言默认与用户需求语言保持一致；若用户未明确指定语言，默认使用中文
- 仅当用户明确要求英文或项目已有强约定要求英文文档时，才输出英文

**中断恢复**：在进入 Step 3 之前**提前确定文件路径**（Step 5 的命名与目录逻辑）。当任意两个维度已有实质内容时，立即写入草稿文件。后续每轮对话更新同一文件，会话中断不会丢失进度。

### 4. 基于模板生成 PRD

收集到足够信息后，读取 PRD 模板生成文档。

**操作步骤**：
1. 用 Read 读取 `.code-flow/specs/shared/prd-template.md`
2. 按照模板章节结构，将 Step 3 中收集的内容填入对应位置
3. **只包含适用的章节**，模板中有但需求不涉及的章节跳过
4. 用 `{}` 占位符替换为实际内容
5. 展示生成的 PRD

**章节映射**：
| 模板章节 | 内容来源 |
|---------|---------|
| 1. 文档控制 | 自动生成 |
| 2. 背景与目标 | Step 3 的"背景与目标" |
| 3. 用户与场景 | Step 3 的"用户与场景" |
| 4. 功能需求 | Step 3 的"功能需求" |
| 5. 非功能需求 | Step 3 的"非功能需求"（按需） |
| 6. 范围与边界 | Step 3 的"范围与边界" |
| 7. 依赖与风险 | Step 3 的"依赖与风险"（按需） |

展示后询问用户：**确认写入？可以调整任何部分。**

### 5. 写入文件

用户确认后：

1. 从目标推导文件名（kebab-case），如 `user-login-flow`
2. 按当前日期创建目录：`.code-flow/tasks/<YYYY-MM-DD>/`
3. 用 Write 写入 `.code-flow/tasks/<YYYY-MM-DD>/<name>.prd.md`

**文件位置说明**：PRD 文档与其衍生的设计简报（`<name>.design.md`）将放在同一目录，方便 `cf-task:archive` 一并归档。

### 6. 输出摘要

```
已生成 PRD: .code-flow/tasks/<日期>/<name>.prd.md

内容:
  - <按实际包含的章节列出>
  - 用户故事: N 个
  - 功能: N 个（P0: X, P1: Y）

下一步:
  - 审阅/编辑: 直接打开 .code-flow/tasks/<日期>/<name>.prd.md
  - 推进设计: /cf-task:align .code-flow/tasks/<日期>/<name>.prd.md
    （align 将从 PRD 派生设计简报，继承目标/用户/功能/范围，只需补技术维度）
```
