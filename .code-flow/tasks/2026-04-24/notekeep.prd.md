## 2. 背景与目标

### 2.1 问题陈述 [必填]

| 维度 | 内容 |
|------|------|
| **问题描述** | 个人笔记需求：现有工具（Notion/Obsidian）太复杂，付费墙高；复制粘贴常带原格式导致排版混乱；缺少本地优先、界面简洁的 MD 笔记工具 |
| **当前替代方案** | Notion（复杂/付费）、Obsidian（学习曲线陡峭）、VS Code + MD 插件（非原生）、Word（不支持 MD） |
| **触发原因** | 个人效率工具需求，寻求轻量、跨平台、本地存储的笔记解决方案 |

### 2.2 目标与价值 [必填]

| 维度 | 内容 |
|------|------|
| **核心目标** | 打造一款轻量、跨平台、界面简洁的本地笔记软件，支持 MD 格式、图片粘贴、日期管理、文件组织 |
| **预期价值** | 降低笔记整理时间；消除复制粘贴格式问题；提升写作体验 |

**成功指标**

| 指标 | 当前值 | 目标值 | 衡量方式 |
|------|--------|--------|----------|
| 冷启动时间 | - | < 2s | 计时 |
| 保存可靠性 | - | 100% 不丢数据 | 压力测试 |
| 用户满意度 | - | > 90% | 问卷 |

## 4. 功能需求

### 4.1 功能清单 [必填]

| 功能ID | 功能名称 | 功能描述 | 优先级 | 来源用户故事 |
|--------|---------|---------|--------|-------------|
| FEAT-01 | MD 编辑器 | 基于 TipTap 的富文本编辑器，支持标准 MD 语法渲染 | P0 | US-08 |
| FEAT-02 | 顶部工具栏 | 类 Word 工具栏：字体、字号、加粗、斜体、下划线、删除线、对齐、列表、引用、代码块、插入图片/链接 | P0 | US-02 |
| FEAT-03 | 文件/文件夹管理 | 新建、删除、重命名文件夹和笔记；树形结构展示 | P0 | US-04, US-09 |
| FEAT-04 | 拖拽排序 | 拖拽调整文件夹/笔记的排列顺序，持久化 | P0 | US-05 |
| FEAT-05 | 按天日记 | 日历视图，按日期筛选/创建日记 | P0 | US-01 |
| FEAT-06 | 图片粘贴 | 监听剪贴板图片，保存到本地并在 MD 中引用 | P0 | US-03 |
| FEAT-07 | 全文搜索 | SQLite FTS5 搜索标题和内容 | P0 | US-06 |
| FEAT-08 | 纯文本粘贴 | 粘贴时自动清除富文本格式，仅保留纯文本 | P0 | US-07 |
| FEAT-09 | 悬浮目录 | 编辑器右侧悬浮目录导航，显示标题层级（H1-H6），点击跳转；5秒无操作自动隐藏 | P1 | - |

#### FEAT-01: MD 编辑器

- **描述**: 基于 TipTap (ProseMirror) 的富文本编辑器，实时渲染标准 MD 语法
- **验收标准**:
  - [ ] 支持 H1-H6 标题
  - [ ] 支持有序/无序列表
  - [ ] 支持代码块（行内代码 + 多行代码块）
  - [ ] 支持引用块
  - [ ] 支持链接和图片

#### FEAT-02: 顶部工具栏

- **描述**: 类 Word 格式工具栏，支持文本格式化操作
- **验收标准**:
  - [ ] 字体选择（预设几种字体）
  - [ ] 字号选择（H1-H6 + 正文）
  - [ ] 加粗、斜体、下划线、删除线
  - [ ] 文本对齐（左/中/右）
  - [ ] 有序列表、无序列表
  - [ ] 引用块、代码块
  - [ ] 插入图片、插入链接

#### FEAT-03: 文件/文件夹管理

- **描述**: 树形结构管理文件夹和笔记
- **验收标准**:
  - [ ] 新建文件夹/笔记
  - [ ] 重命名（单击两次或右键菜单）
  - [ ] 删除（递归删除子项）
  - [ ] 移动到其他文件夹（拖拽或右键）

#### FEAT-04: 拖拽排序

- **描述**: 通过拖拽调整同级项的排列顺序
- **验收标准**:
  - [ ] 同级文件夹可拖拽排序
  - [ ] 同级笔记可拖拽排序
  - [ ] 排序结果持久化到数据库

#### FEAT-05: 按天日记

- **描述**: 日历视图，按日期查看/创建日记
- **验收标准**:
  - [ ] 月视图日历展示
  - [ ] 点击日期查看当天日记列表
  - [ ] 无日记时快速创建
  - [ ] 笔记自动关联日期（可手动修改）

#### FEAT-06: 图片粘贴

- **描述**: 粘贴图片时自动保存并插入 MD 引用
- **验收标准**:
  - [ ] 监听 `paste` 事件检测图片
  - [ ] 图片保存到 `~/.notekeep/data/images/`（相对路径支持迁移）
  - [ ] 生成 UUID 文件名避免冲突
  - [ ] MD 中插入 `![](images/uuid.png)` 格式（相对路径）

#### FEAT-07: 全文搜索

- **描述**: SQLite FTS5 虚拟表实现全文搜索
- **验收标准**:
  - [ ] 搜索标题和内容
  - [ ] 搜索结果高亮匹配词
  - [ ] 支持按日期范围筛选

#### FEAT-08: 纯文本粘贴

- **描述**: 粘贴富文本时自动清除格式，仅保留纯文本
- **验收标准**:
  - [ ] 监听 `paste` 事件
  - [ ] 提取纯文本内容替换
  - [ ] 保留换行符

#### FEAT-09: 悬浮目录

- **描述**: 编辑器右侧悬浮目录导航，方便长文档内快速跳转
- **验收标准**:
  - [ ] 显示当前笔记的标题层级（H1-H6）
  - [ ] 点击目录项跳转到对应位置
  - [ ] 滚动时自动显示
  - [ ] 5 秒无操作自动隐藏

### 4.2 边缘情况 [可选]

| 场景 | 预期行为 |
|------|----------|
| 删除含子项的文件夹 | 递归删除所有子笔记，弹出确认框 |
| 粘贴非图片内容 | 正常粘贴文本，不触发图片处理 |
| 搜索无结果 | 显示"未找到相关笔记"提示 |
| 图片文件丢失 | 显示占位符，提示图片不存在 |
| 笔记内容为空 | 自动保存空内容，不报错 |

## 6. 范围与边界 [必填]

| 类别 | 内容 |
|------|------|
| **范围（In Scope）** | 桌面端笔记软件（Tauri + React）；本地 SQLite 存储；MD 编辑器；图片粘贴；文件管理；日历日记；全文搜索 |
| **非范围（Out of Scope）** | 云同步/多设备；移动端；协作功能；插件系统；导出为 PDF/Word |
| **前置假设** | 用户有本地文件系统读写权限；图片单文件 < 10MB |

**数据迁移**: `~/.notekeep/` 为自包含数据目录，整体复制即可迁移。数据库内图片路径使用相对路径 `./images/xxx.png`，确保迁移后仍可正确解析。
